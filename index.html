Laboratory Exercises

Lab 1: User Interface Layouts

Constructing custom layouts, alignments, and basic components using declarative UI structures.

Reflection

I learned how to shift from declarative XML schemas to modular Jetpack Compose functions. Setting up clean column and row architectures allowed me to understand precise padding controls. This laid the foundation for pixel-perfect structural layouts.

Lab 2: State & Interaction

Managing screen state and component interaction using mutable states and event hoisting handlers.

Reflection

This lab introduced the core concepts of state handling using remember and mutableStateOf primitives. I learned how recomposition alters UI trees upon data modifications. Mastering state hoisting made interactive dynamic adjustments highly structured.

Lab 3: Material Design 3 Systems

Implementing standard themes, dark/light modes, color schemes, and typographies.

Reflection

I applied Material Design 3 design system tokens to customize shapes, surfaces, and typography groupings. Configuring localized color pallets enhanced consistency across active components. This exercise established proper application theme control.

Lab 4: App Jetpack Navigation

Structuring explicit cross-screen routing workflows via declarative NavHost engines.

Reflection

Setting up NavHost components and passing safe parameter keys allowed me to create cohesive multi-screen applications. Managing backstack entries prevented screen duplication memory bugs. This navigation flow completely streamlined complex user journeys.

Lab 5: Room Architecture Database

Persisting continuous structured offline user models inside hardware storage sandboxes.

Reflection

Integrating underlying SQLite infrastructures using Room DAOs allowed me to safely persist asynchronous data streams via Coroutines. Mapping structural local schemas directly automated data binding requirements. This exercise secured full localized client data durability.
